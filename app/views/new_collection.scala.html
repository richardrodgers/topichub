Define a content collection

All the content you publish to the hub must be placed in a collection. You may define as many collections as you like, but items in a given collection must share the first two attributes on the left: a content type and a package format. If your content cannot be described by one of these choices, you may not deposit it to the hub. Package format names abbreviate the kinds of files in a package; so mets-nlm means that in addition to an article PDF, the package has a METS file and a metadata file conforming to the NLM-DTD schema.

The description field is the collection name, and the policy describes any use or other restrictions you wish to express. This policy will appear, e.g. in the SWORD service document for the collection.

When you have saved the collection definition, the hub will email you precise instructions on how to make deposits to it.
